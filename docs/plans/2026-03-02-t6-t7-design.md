# T6 + T7 Design: Health Alerts + Error Resilience

## Overview

Make the always-on worker bulletproof: every job catches its own exceptions, fires
specific Telegram alerts, and the LLM extraction job is protected by a circuit breaker.
A heartbeat job detects staleness (no poll in >30 min).

---

## Architecture

Option A (thin wrappers) selected over decorator-based or APScheduler event listeners.
Explicit try/except per job allows type-specific alert messages rather than generic ones.

---

## New Files

### `packages/core/src/core/health.py`

In-memory rate-limited alert dispatcher.

```python
_last_alert: dict[str, datetime] = {}

def alert(key: str, message: str, level: str = "warning", cooldown_minutes: int = 30) -> None
```

- `key` deduplicates: same key suppressed within cooldown window
- Never raises — wraps `send_message()` in try/except
- Emoji map: warning=⚠️, error=🔴, info=ℹ️

### `packages/core/src/core/circuit_breaker.py`

In-memory circuit breaker for LLM extraction.

```python
class CircuitBreaker:
    def __init__(self, name: str, threshold: int = 5, reset_minutes: int = 10)
    def record_failure(self) -> None   # trips at threshold
    def record_success(self) -> None   # resets counter + opens circuit
    def is_open(self) -> bool          # True = skip, False = allow
```

- State is **in-memory only** — resets on worker restart (intentional)
- One module-level instance: `llm_breaker = CircuitBreaker("llm")`

## Changes to Existing Files

### `apps/worker/src/worker/jobs.py`

1. Add `_last_poll: dict[str, datetime]` module-level dict — updated on successful poll
2. Wrap every job in `try/except` with type-specific `alert()` calls
3. Add `job_heartbeat()` — checks `_last_poll` staleness (>30 min), fires alert

### `apps/worker/src/worker/main.py`

Add one line:
```python
scheduler.add_job(job_heartbeat, IntervalTrigger(minutes=5), id="heartbeat")
```

## Alert Inventory

| Key                  | Trigger                              | Message |
|----------------------|--------------------------------------|---------|
| `gmail_auth`         | GmailAuthError in poll               | Gmail auth expired. Run: `claw connect gmail` |
| `outlook_auth`       | OutlookAuthError in poll             | Outlook auth expired. Run: `claw connect outlook` |
| `gmail_poll_error`   | Generic exception in Gmail poll      | Gmail poll failed: {exc} |
| `outlook_poll_error` | Generic exception in Outlook poll    | Outlook poll failed: {exc} |
| `gcal_poll_error`    | Generic exception in GCal poll       | GCal poll failed: {exc} |
| `llm_circuit_open`   | Circuit breaker trips                | LLM extraction paused (5 consecutive failures). Will retry in 10 min. |
| `llm_circuit_reset`  | Circuit resets after backoff         | LLM extraction resumed. |
| `stale_gmail`        | No Gmail poll in >30 min             | No Gmail sync in 30+ min — worker may have crashed. |
| `stale_outlook`      | No Outlook poll in >30 min           | No Outlook sync in 30+ min. |
| `extract_job_error`  | Generic exception in extract job     | Extraction job failed: {exc} |
| `reminder_job_error` | Generic exception in reminders job   | Reminder dispatch failed: {exc} |

---

## Design Decisions

- **In-memory circuit breaker** (not DB-persisted): worker restart = natural recovery signal
- **Rate-limit by alert key** (not every failure): prevents Telegram spam on sustained outages
- **Module-level `_last_poll` dict** in jobs.py for staleness detection (not DB)
- **Cooldown default: 30 min** — enough to notify once per sustained outage, not per job run
